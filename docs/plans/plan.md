# Plan: Zero-Knowledge Conditional Entropy for zkLLM

## Overview

Add conditional entropy computation to zkLLM to bound covert information capacity in LLM inference outputs. This enables proving an upper bound on how much hidden information could be encoded in model outputs.

## Background

Given a claimed model output sequence, we want to compute:

$$\hat{C}(O) = -\sum_{i=1}^{n} \log_2 q_i(o_i)$$

Where $q_i$ is the probability distribution over tokens at position $i$, accounting for hardware noise in the logits.

This bounds covert channel capacity: any steganographic information in the output cannot exceed this entropy bound.

## Approach

1. Use temperature zero (greedy decoding) to avoid Gumbel-Max seed complexity
2. Model hardware noise as Gaussian on logit differences
3. Compute win probability for each token under noise model
4. Normalize to get probability distribution $q$
5. Compute surprise $-\log_2 q(o_i)$ for each actual output token
6. Sum over sequence for total conditional entropy bound

## Implementation Plan

> **File location note**: All new files go directly in `zkllm-ccs2024/` alongside the existing `.cu`/`.cuh` files — there is no `src/` subdirectory.

### Phase 1: Argmax Verification

**File: `zkllm-ccs2024/zkargmax.cuh`**

Create header with class declaration:

```cpp
class zkArgmax {
public:
    zkArgmax(uint bit_width);

    // Compute argmax (plain computation, no proof)
    uint compute(const FrTensor& logits);

    // Prove that t_star is the argmax of logits.
    // Returns the multilinear extension claim on logits at the verifier's challenge
    // point, so the upstream sum-check can be linked.
    Fr_t prove(
        const FrTensor& logits,
        uint t_star,
        Fr_t v_star,
        const vector<Fr_t>& u,
        vector<Polynomial>& proof
    );

private:
    uint bit_width;
};
```

**File: `zkllm-ccs2024/zkargmax.cu`**

Implement using Option A (difference tensor with bit decomposition):

- Compute `diffs[i] = v_star - logits[i]` for all i
- Bit-decompose each diff into `bit_width` bits
- Prove each bit is binary: `bit * (bit - 1) = 0`
- Prove reconstruction: `diffs[i] = sum_b(bits[i,b] * 2^b)`
- Prove `logits[t_star] = v_star` via indexed lookup
- Use existing FrTensor operations and proof infrastructure from zkLLM.

### Phase 2: Log Lookup Table

**File: `zkllm-ccs2024/zklog.cuh`**

Create header:

```cpp
class zkLog {
public:
    // precision: bits of precision for input probabilities
    // scale: output scaling factor
    zkLog(uint precision, uint scale);

    // Compute -log2(p) for each element
    FrTensor compute(const FrTensor& probs);

    // Prove the log computation is correct.
    // Returns the MLE claim on (probs, log_probs) at the verifier's challenge point.
    Fr_t prove(
        const FrTensor& probs,
        const FrTensor& log_probs,
        const FrTensor& m,           // multiplicity counts from prep()
        const Fr_t& r, const Fr_t& alpha, const Fr_t& beta,
        const vector<Fr_t>& u, const vector<Fr_t>& v,
        vector<Polynomial>& proof
    );

private:
    FrTensor table_input;   // Probability values
    FrTensor table_output;  // Corresponding -log2 values
    uint precision;
    uint scale;
};
```

**File: `zkllm-ccs2024/zklog.cu`**

Implement as a `tLookupRangeMapping` (see `tlookup.cuh`):

- Precompute lookup table: for each quantized probability index p ∈ [0, 2^precision), store -log2(p / 2^precision) * scale
- Input quantized probs are in range [0, 2^precision); output is scaled log value
- `compute()` calls `tLookupRangeMapping::operator()` to get `(log_probs, m)` in one step
- `prove()` delegates to `tLookupRangeMapping::prove(S_in, S_out, m, r, alpha, beta, u, v, proof)`
- The random scalars `r, alpha, beta` and challenge vectors `u, v` are generated by the caller (same pattern as `zkSoftmax::prove`)

### Phase 3: Normal CDF Lookup Table

**File: `zkllm-ccs2024/zknormalcdf.cuh`**

Create header:

```cpp
class zkNormalCDF {
public:
    // precision: bits of precision for input
    // scale: output scaling factor
    // sigma: standard deviation for the normal distribution
    zkNormalCDF(uint precision, uint scale, Fr sigma);

    // Compute Phi((v_star - logits[i]) / (sigma * sqrt(2)))
    FrTensor compute(const FrTensor& diffs);

    // Prove the CDF computation is correct.
    // Same signature convention as zkLog::prove.
    Fr_t prove(
        const FrTensor& diffs,
        const FrTensor& cdf_values,
        const FrTensor& m,
        const Fr_t& r, const Fr_t& alpha, const Fr_t& beta,
        const vector<Fr_t>& u, const vector<Fr_t>& v,
        vector<Polynomial>& proof
    );

private:
    FrTensor table_input;
    FrTensor table_output;
    uint precision;
    uint scale;
    Fr sigma;
};
```

**File: `zkllm-ccs2024/zknormalcdf.cu`**

Implement as a `tLookupRangeMapping` (same pattern as zkLog):

- Table covers signed range `[-2^(precision-1), 2^(precision-1))` representing scaled logit diffs
- For each index d, store `Phi(d / (sigma * sqrt(2))) * scale` as the output
- Negative diffs are represented as `p - |d|` in the field (standard convention); the table low/len parameters handle the signed range
- `compute()` and `prove()` follow the same pattern as zkLog

### Phase 4: Conditional Entropy Calculator

**File: `zkllm-ccs2024/zkentropy.cuh`**

Create header:

```cpp
class zkConditionalEntropy {
public:
    zkConditionalEntropy(
        uint vocab_size,
        uint bit_width,      // Quantization bits for logits
        uint precision,      // Precision for probability calculations
        Fr sigma             // Calibrated hardware noise std
    );

    // Compute conditional entropy for a sequence
    // Returns sum of -log2(q[actual_token]) over all positions
    Fr compute(
        const vector<FrTensor>& logits_sequence,  // Logits at each position
        const vector<uint>& actual_tokens          // Actually selected tokens
    );

    // Prove the entropy computation.
    // Returns the sum of per-position MLE claims for linking to upstream logit proofs.
    Fr_t prove(
        const vector<FrTensor>& logits_sequence,
        const vector<uint>& actual_tokens,
        Fr_t claimed_entropy,
        vector<Polynomial>& proof
    );

private:
    zkArgmax argmax;
    zkNormalCDF normal_cdf;
    zkLog log_lookup;
    uint vocab_size;
    Fr sigma;
};
```

**File: `zkllm-ccs2024/zkentropy.cu`**

Implement the full pipeline for each position. All element-wise operations on `FrTensor` data must be CUDA kernels (the data lives on GPU); there is no CPU-side `[]` indexing. The sketch below uses pseudocode for GPU steps — each becomes a `__global__` kernel or a call to an existing `FrTensor` operation:

```
computePosition(logits, actual_token, proof):

  1. Argmax
     t_star  = argmax.compute(logits)         // CPU scalar
     v_star  = logits(t_star)                 // copy single Fr_t to host
     u       = random_vec(ceilLog2(vocab_size))
     argmax.prove(logits, t_star, v_star, u, proof)  // returns MLE claim

  2. Compute diffs (GPU kernel: diffs[i] = v_star - logits[i])
     FrTensor diffs = broadcast_sub(v_star, logits)   // use Fr_broadcast_sub

  3. Win probabilities via normal CDF (tLookupRangeMapping)
     Quantize diffs to signed integer indices (GPU kernel, same convention as zkrelu)
     auto [win_probs, m_cdf] = normal_cdf(diffs_quantized)
     normal_cdf.prove(diffs_quantized, win_probs, m_cdf, r, alpha, beta, u, v, proof)

  4. Normalize
     Fr_t total     = win_probs.sum()          // existing FrTensor reduction
     Fr_t total_inv = Fr_t::inverse(total)     // field inverse
     FrTensor q     = win_probs * total_inv    // broadcast multiply (GPU kernel)
     // Prove: append (total, total_inv, claim that total*total_inv = 1) to proof

  5. Quantize q for log lookup
     // q values are field elements representing fractions in [0,1).
     // Re-scale to integer indices in [0, 2^precision) via GPU kernel:
     //   q_idx[i] = round(q[i] * 2^precision / total_scale)
     // where total_scale accounts for the fixed-point scaling factor used in step 3.
     // Extract q_idx[actual_token] as a single-element FrTensor q_actual.

  6. Surprise via log lookup (tLookupRangeMapping)
     auto [surprise, m_log] = log_lookup(q_actual)
     log_lookup.prove(q_actual, surprise, m_log, r2, alpha2, beta2, u2, v2, proof)

  return surprise[0]   // copy single Fr_t to host
```

**Key implementation note — quantization bridge (steps 3→4→5):**
`win_probs` from the CDF lookup are scaled integers (e.g., multiplied by `scale_cdf`). After summing and dividing, `q[i]` is a field element representing `(win_probs[i] / total) * scale_cdf`. To feed into the log table, convert each `q[i]` to an index in `[0, 2^precision)` by computing `round(q[i] * 2^precision / scale_cdf)` in a GPU kernel. This re-quantization step must itself be range-proved (bit decomposition of the rounding remainder, same pattern as `zkrelu`).

### Phase 5: Integration with zkLLM Pipeline

**File: `zkllm-ccs2024/zkllm_entropy.cu`**

The existing `main.cu` is a stub (the model loading is not yet implemented). Phase 5 should therefore be a standalone binary that:
1. Reads pre-saved logit tensors from disk (one `FrTensor` file per output position, produced by an earlier zkLLM run)
2. Reads the actual output token sequence
3. Runs `zkConditionalEntropy::compute` then `::prove`
4. Writes the proof and entropy bound to disk

```cpp
int main(int argc, char* argv[]) {
    // argv[1]: directory of saved logit FrTensor files (logits_0.bin, logits_1.bin, ...)
    // argv[2]: token sequence file (one uint per line)
    // argv[3]: proof output path
    // argv[4]: calibrated sigma (float, passed as string)

    string logits_dir    = argv[1];
    string tokens_file   = argv[2];
    string proof_output  = argv[3];
    double sigma_val     = atof(argv[4]);

    // Load tokens
    vector<uint> output_tokens = load_token_sequence(tokens_file);
    uint T = output_tokens.size();

    // Load logit tensors (use existing FrTensor serialization from ioutils)
    vector<FrTensor> logits_sequence;
    for (uint t = 0; t < T; t++) {
        logits_sequence.push_back(FrTensor::load(logits_dir + "/logits_" + to_string(t) + ".bin"));
    }

    uint vocab_size = logits_sequence[0].size;

    zkConditionalEntropy entropy_prover(
        vocab_size,
        16,                          // bit_width for argmax range proof
        16,                          // precision for log/CDF tables (2^16 entries)
        double_to_scalar(sigma_val)  // calibrated hardware noise sigma
    );

    vector<Polynomial> proof;
    Fr_t total_entropy = entropy_prover.compute(logits_sequence, output_tokens);
    entropy_prover.prove(logits_sequence, output_tokens, total_entropy, proof);

    // Serialize proof (use existing pattern from proof.cuh)
    save_proof(proof, proof_output);

    long entropy_bits = scalar_to_long(total_entropy);  // scaled fixed-point
    cout << "Conditional entropy bound: " << entropy_bits << " (scaled)" << endl;
    cout << "Average per token: " << entropy_bits / T << endl;

    return 0;
}
```

**Note**: `FrTensor::load` and `save_proof` — check `ioutils.cuh` for the exact serialization API and use it directly rather than inventing a new format.

## Key Implementation Notes

- **Use existing primitives**: FrTensor, Polynomial, commitment, tlookup patterns are all in the codebase. Don't reinvent.
- **Follow existing code style**: Look at `zksoftmax.cu` and `zkfc.cu` for patterns on how to structure compute/prove pairs. Prove functions return `Fr_t` (the MLE claim at the verifier's challenge point) so they can be chained.
- **All FrTensor ops are GPU kernels**: There is no CPU-side element access on FrTensor (data lives in `gpu_data`). Loops like `for i: diffs[i] = ...` must be written as `__global__` CUDA kernels, or use existing `Fr_broadcast_sub` / `Fr_elementwise_*` kernels from `fr-tensor.cuh`.
- **Bit decomposition**: For range proofs, decompose into bits and prove each bit is binary via `bit * (bit - 1) = 0`. This is standard.
- **tlookup for nonlinear functions**: Both log and normal CDF use `tLookupRangeMapping` from `tlookup.cuh`. The `operator()(FrTensor)` call returns `(output_values, multiplicity_counts)` in one step. The `prove()` method takes `(S_in, S_out, m, r, alpha, beta, u, v, proof)` — match this signature exactly.
- **Quantization bridge**: When passing values from one lookup to the next (e.g., normalized `q` → log lookup), re-quantize in a GPU kernel and range-prove the rounding remainder with bit decomposition (same as `zkrelu`). Track the scaling factor through each step.
- **Field representation of negatives**: Negative values are represented as `p - |x|` in the field. The sign convention should match existing zkLLM code (see `zkrelu.cu` for the signed decomposition pattern).
- **Proof challenge vectors**: `r`, `alpha`, `beta`, `u`, `v` are generated by the caller with `random_vec()` and passed in. Each sub-prover does not generate its own randomness.

## Calibration (Pre-implementation)

Before implementation, empirically calibrate:

1. Run the target model twice on same inputs with temperature 0
2. Measure what fraction of tokens match exactly
3. Find sigma such that the noise model predicts this match rate
4. Hardcode calibrated sigma or make it a parameter

Target: ~97% exact match rate under the noise model.

## Success Criteria

- `zkargmax` correctly proves argmax for arbitrary logit tensors
- `zklog` correctly proves -log2 computation
- `zknormalcdf` correctly proves normal CDF computation
- `zkentropy` produces valid proofs that verify
- Entropy bound is tight: greedy decoding of likely tokens gives low entropy
- Entropy bound catches anomalies: unlikely tokens give high entropy
- Total overhead is <10% of base zkLLM proving time
